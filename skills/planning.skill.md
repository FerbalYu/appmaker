# 策划技能

定义如何理解需求、拆解任务、制定执行计划。

## 输入

| 类型       | 说明              | 示例                                 |
| ---------- | ----------------- | ------------------------------------ |
| 自然语言   | 用户描述的需求    | "做一个博客系统，支持文章发布、评论" |
| 结构化需求 | 已有明确规格      | API 文档、设计稿                     |
| 既有代码   | 需维护/优化的项目 | 已有项目路径                         |

## 流程

```
用户需求
    ↓
┌─────────────────────────────────────────────────────────────┐
│  阶段 1：理解需求                                            │
│  - 提取关键实体（用户、功能、系统）                          │
│  - 识别核心目标与次要目标                                    │
│  - 澄清模糊点（如有）                                        │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  阶段 2：分解任务                                            │
│  - 生成任务树（树形结构）                                    │
│  - 每个叶子节点是一个可执行任务                              │
│  - 标注任务类型（创建/修改/测试/文档）                        │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  阶段 3：评估依赖                                            │
│  - 任务间依赖关系                                            │
│  - 资源需求（token 预算）                                   │
│  - 预估执行时间                                              │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  阶段 4：生成里程碑                                          │
│  - 划分阶段（Phase 1: 核心功能...）                          │
│  - 每个里程碑有明确的交付物                                  │
│  - 设置检查点                                                │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  阶段 5：输出执行计划                                        │
│  - 结构化 JSON 格式                                          │
│  - 包含完整任务列表和依赖图                                  │
│  - 供 execution.skill 使用                                  │
└─────────────────────────────────────────────────────────────┘
```

## 输出格式

```json
{
  "plan_id": "uuid",
  "created_at": "ISO timestamp",
  "project": {
    "name": "项目名称",
    "description": "项目描述"
  },
  "milestones": [
    {
      "id": "m1",
      "name": "里程碑 1：核心框架",

      "deliverables": ["文件1", "功能1"]
    }

      "id": "t1",

  ],

    "total_tasks": 10,
    "total_tokens_estimate": 50000,
    "total_minutes_estimate": 120
  }
}
```

## 任务树示例

```
博客系统
├── 阶段 1：基础框架
│   ├── t1: 初始化项目结构
│   ├── t2: 配置数据库连接
│   └── t3: 实现基础模型
├── 阶段 2：文章功能
│   ├── t4: 文章 CRUD API
│   ├── t5: 富文本编辑器集成
│   └── t6: 文章列表页
├── 阶段 3：评论功能
│   ├── t7: 评论 API
│   ├── t8: 评论显示组件
│   └── t9: 评论通知
└── 阶段 4：上线准备
    ├── t10: 部署配置
    └── t11: 性能测试
```

## 约束规则

1. 每个任务必须可独立验证
2. 依赖关系必须无环
3. 预估时间用于资源管理
4. 任务描述必须清晰可执行

## 决策原则

| 情况     | 决策                       |
| -------- | -------------------------- |
| 需求模糊 | 先澄清，再分解             |
| 任务太大 | 继续拆分到 30 分钟内可完成 |
| 依赖复杂 | 优先安排独立任务并行       |
| 资源有限 | 先做核心功能，延后次要功能 |
