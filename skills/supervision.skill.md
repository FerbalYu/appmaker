# 监督技能

定义如何监控进度、检查质量、管理资源、触发修正。

## 监控维度

```
┌─────────────────────────────────────────────────────────────────┐
│                        监督 Dashboard                            │
├─────────────────┬─────────────────┬─────────────────────────────┤
│     进度        │      质量       │           资源               │
├─────────────────┼─────────────────┼─────────────────────────────┤
│ 已完成/总任务   │ 代码规范检查     │ Token 消耗 / 预算            │
│ 当前里程碑     │ 测试覆盖率       │ API 调用次数                 │
│ 预计剩余时间   │ Bug 数量         │ 运行时间                     │
│ 偏离计划？     │ 架构违规？       │ 成本超支？                   │
└─────────────────┴─────────────────┴─────────────────────────────┘
```

## 监控指标

| 指标       | 正常范围   | 警告阈值     | 危险阈值     |
| ---------- | ---------- | ------------ | ------------ |
| 任务完成率 | -          | 低于预期 20% | 低于预期 50% |
| 质量得分   | ≥ 85%      | 70-84%       | < 70%        |
| Token 消耗 | < 80% 预算 | 80-95% 预算  | > 95% 预算   |
| 错误累积   | < 3        | 3-5          | > 5          |
| 计划偏离   | 0          | 轻微偏离     | 严重偏离     |

> **注意**：质量得分阈值已提升至 85，低于此分数的代码将被要求修正。

## 检查点 (Checkpoints)

### 自动检查点

- 每个里程碑结束时
- 每 10 个任务完成
- Token 消耗达到 50%

### 检查内容

```json
{
  "checkpoint_id": "cp_001",
  "timestamp": "ISO timestamp",
  "milestone": "m2",
  "stats": {
    "tasks_completed": 5,
    "tasks_failed": 1,
    "total_time_minutes": 45,
    "tokens_spent": 25000,
    "errors": ["e1", "e2"]
  },
  "quality": {
    "lint_passed": true,
    "tests_passed": true,
    "coverage": "75%"
  },
  "risks": ["资源消耗较快", "1个任务失败"]
}
```

## 异常处理流程

```
检测到异常
    │
    ▼
┌─────────────────────┐
│ 判断异常类型         │
├─────────────────────┤
│ - 任务失败          │
│ - 质量不达标        │
│ - 资源超限          │
│ - 计划偏离          │
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│ 查询 self-correction │
│ .rules.md 触发条件   │
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│ 触发修正流程         │
│ 或                  │
│ 人工介入（严重时）   │
└─────────────────────┘
```

## 风险评估

| 风险等级 | 条件          | 处理方式             |
| -------- | ------------- | -------------------- |
| 🟢 低    | 所有指标正常  | 继续执行             |
| 🟡 中    | 1-2 个警告    | 记录，增加监控频率   |
| 🔴 高    | 危险阈值触发  | 触发 self-correction |
| ⛔ 严重  | 多重危险/崩溃 | 暂停，等待人工决策   |

## 日志规范

监督层维护的日志：

```
/logs/
├── execution/
│   ├── task_t1.log    # 单个任务执行日志
│   ├── task_t2.log
│   └── ...
├── quality/
│   ├── lint_001.log  # 质量检查日志
│   └── test_001.log
├── supervisor.log    # 主监督日志
└── corrections/       # 修正记录
    ├── corr_001.md
    └── corr_002.md
```
